The core idea

An embedding is a point in meaning space, not a string

Same model embeds the query and every chunk. "Retrieve" = find the nearest points. Near in space ⇒ near in meaning — if the model learned meaning.
RAG 103 · What an embedding is

Why a real model, not a bigger dumb one

A dimension only helps if it means something

Hashing (dumb)

Each dim = an arbitrary bucket words hash into. "home" → bucket 88, "lair" → bucket 203. Synonyms scatter. More buckets (256→4096) just spreads them further — capacity isn't comprehension.

Learned (real)

Each dim = an abstract feature shaped by training. "home" and "lair" land almost on top of each other. Coordinates encode meaning. The win is meaningful axes, not more axes.

102 → 103 is "256 dumb dims → 768 learned dims." The jump is the meaning, not the count.

Measuring "near"

Cosine compares the angle — normalize, and length stops mattering

Raw L2 distance ? is fooled by length: a long on-topic chunk has a big-magnitude vector and ranks too low.

Cosine = angle only, ignores magnitude. Normalize every vector to length 1 (÷ its own length) and they all sit on the unit sphere — where L2 ranking == cosine ranking.

So a chunk isn't penalized for being wordy. Direction (meaning) decides, not size.

What a vector DB actually does

Brute-force is exact; ANN trades recall for speed

Brute force: compare the query to every vector, sort, take top-K. Always correct, but O(N) per query.
ANN (HNSW / IVF) ?: prebuilt index, checks a small neighborhood. Fast, but may miss the true nearest — a recall ⇄ latency dial.

Our choice: sqlite-vec does exact brute-force KNN — no ANN index. At 48 chunks (and a full DDB import) that's the right call: exact, zero tuning, simpler.

Approximate is fine in RAG anyway — we over-fetch top-K and a reranker (105) trims it. ANN only earns its keep at large N with a tight latency budget. The Store seam stays swappable if we outgrow it.

A constraint with teeth

Vectors are a derived cache — change the model, re-ingest everything

The store's vector column is fixed-width (float[768]). You can't compare a 768-dim query to 256-dim stored vectors — the math is component-wise, and two different models have unrelated coordinate systems anyway.

So dim is one-way per DB. Vectors = a function of (corpus + model version). Upgrade the model ⇒ invalidate the cache ⇒ re-embed from source. No migration path.
The build hit this live. A leftover 256-dim DB + CREATE TABLE IF NOT EXISTS silently kept the old dim → the 768-dim insert died with an opaque Dimension mismatch mid-transaction.
Fix (fail loud): store.init() now reads the existing dim and throws a clear "wipe + re-ingest" message before any work — not an auto-wipe (never silently delete the user's data).

Gain

Cost & privacy

No per-call fees; the corpus never leaves the machine (DDB copyright + the Neumo rule). $0 vs a $500-overnight cloud bill.

Gain

Repeatability

Pin the model → stable vectors. A hosted model being deprecated would silently force a full re-ingest.

Give up

Quality ceiling

Top hosted models out-rank small local ones; plus you run the ops. Benchmark hosted before a quality-critical prod.

What real embeddings don't fix

The surviving intruder is honest — and it's 104's job

#2 Deep Gnome ("svirfneblin… deep underground") and the Giants cluster all carry cave/underground semantics. A habitat query genuinely resembles every "lives underground" monster.

That's the synonym win working, not a bug. Semantic search did exactly its job.
But the literal token goblin would pull goblin chunks decisively. Meaning-similarity can't out-rank an exact identifier on its own.

This is the lexical/identifier gap — the concrete, observed motivation for 104: hybrid retrieval (dense + sparse/BM25, fused). These exact ranks are 104's "before."

Key takeaways

What to walk away with

  • An embedding is a point in meaning space — a learned model puts synonyms near each other; hashing can't.
  • Normalize, then cosine — angle = meaning; length shouldn't decide. On the unit sphere, L2 == cosine.
  • A vector DB does (A)NN search — brute-force is exact and right at our scale; ANN is a recall/latency dial for later.
  • Vectors are a derived cache — change the model/dim ⇒ full re-ingest; the build made that a fail-loud guard, not a footnote.
  • Verified: MRR 0.33 → 1.0, neighbors fixed. The one honest intruder is the lexical gap → 104.
