What we're building

A support-refund agent you can actually run

An agent handles a customer support ticket and decides whether to issue a refund. We send it a normal ticket and a poisoned one, and watch the architecture — not the model's good behavior — keep us safe.

Goal isn't "a clever agent." It's to demonstrate containment: even when the model is fooled, the deterministic gate refuses the dangerous action.

MCP 103 · Architecture

Piece 1 · the mock platform

Flat JSON — reads real, writes simulated

The "billing platform" is a JSON file plus a small store.ts data layer.

  • Reads are real: getOrder, getTicket read from the file.
  • Writes are simulated: issueRefund logs and returns a fake receipt — it never mutates data.
Why simulate writes? The whole lesson is about the decision to act, not the side effect. Faking the write keeps the demo safe, deterministic, and dependency-free.
MCP 103 · Mock platform

Piece 2 · the MCP server

Three tools, each with a trust level

Untrusted

get_ticket

Returns customer-written text. Output is tagged ☣ tainted at the boundary.

Trusted

get_order

Returns our own system-of-record order data. The gate trusts this.

Privileged

issue_refund

The dangerous action. Gated; only fires if policy allows.

Built on the official @modelcontextprotocol/sdk over stdio, with zod input schemas. The trust level is declared metadata — that's what the orchestrator's taint layer keys off.
MCP 103 · MCP server

Piece 3 · the orchestrator

102's defense, now in code

  • Planner LLM — sees only the trusted request; emits the plan and drives the read tools. It proposes the refund; it never calls the privileged tool directly.
  • Quarantine LLM — no tools; turns the tainted ticket prose into a typed RefundRequest.
  • Taint layer — wraps untrusted-derived values so provenance can't be lost.
  • Policy gate — plain code; the only thing that can authorize issue_refund.
Wired together with a LangGraph StateGraph: the gate is a conditional edge driven by code, not by model text.
MCP 103 · Orchestrator

The LangGraph flow

Nodes do work; the edge makes the decision

The branch keys off the gate's structured boolean result — never the model's free text. That's the rule from 102 made literal.
MCP 103 · LangGraph flow

Code · taint tracking

Provenance you can't accidentally drop

// a value + where it came from
interface Tainted<T> {
  readonly __tainted: true;
  readonly source: string;  // "ticket:tkt_666"
  readonly value: T;
}

// loud on purpose — grep for reveal(
function reveal<T>(t: Tainted<T>): T {
  return t.value;
}

The brand makes it awkward to pass tainted data where plain data is expected — you must consciously reveal() it.

Every reveal( is a trust boundary you can audit by grep. Taint is sticky: derive anything from it and the result is tainted too.
MCP 103 · Taint code

Code · the policy gate

You cannot prompt-inject an if

// default DENY; ALLOW only if all checks pass
if (order.customerId !== ticketCustomerId)
  return escalate("cross-account refund");

if (req.amount > REFUND_CAP)        // $50
  return escalate("over auto-approve cap");

if (req.amount > order.amount)
  return escalate("exceeds order total");

return { decision: "ALLOW" };

The attack's $9,999 → cust_999 fails check one (ownership) and check two (cap). It never reaches issue_refund.

The injected prose may fool the quarantine LLM into emitting a bad value — but it has zero power over this boolean logic. Bad value, never bad action.
MCP 103 · Policy code

What you see when you run it

Same pipeline, two outcomes

tkt_123 · benign

Quarantine extracts {ord_123, $42} (tainted) → gate: owner ✓, ≤ cap ✓ → ALLOW → simulated $42 receipt.

tkt_666 · injection

Quarantine (fooled) extracts {$9999, cust_999} (tainted) → gate: owner ✗, cap ✗ → ESCALATE → no refund.

The planner never saw the injected text; the parser had no tools; the gate refused the action. Three independent reasons the attack failed — defense in depth.

Honesty about the demo

What's real, what's faked

Real

  • MCP server + tool definitions & trust levels
  • LangGraph state machine & the conditional edge
  • Taint propagation and the policy gate logic
  • LLM planner/quarantine (with an API key)

Simulated

  • Billing data is a small JSON file, not a real platform
  • Refund writes are logged, not persisted
  • The browser version scripts the LLM steps (no key, no network)

Where each 102 idea lives

Concept → file

  • Taint / provenance → src/orchestrator/taint.ts
